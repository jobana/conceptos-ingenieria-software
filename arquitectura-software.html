Arquitectura de Software

La arquitectura es un conjunto de modelos altamente abstractos. Estos modelos tienen la función de abordar, comprender y resolver problemas complejos que implican la participación de múltiples dominios a la vez. Esencialmente, la arquitectura representa la ingeniería llevada a un nivel superior de abstracción, cuyo resultado son modelos que no se implementan de forma directa.

Conceptos Clave

Dominio

Punto de vista o dimensión que participa en el problema o la solución. Ayuda a descomponer la complejidad.

Modelo

Abstracción o plano del problema/solución. Sirve para entender y diseñar antes de implementar.

Solución

Respuesta integral implementada a un problema o necesidad. Incluye tecnología y otros elementos (ej. procesos, negocio).

  • Estilos Arquitectónicos

Monolito

Aplicación en un solo bloque

Toda la aplicación en un solo bloque. Arquitectura tradicional donde todos los componentes están integrados en una sola aplicación. Simple de desarrollar pero difícil de escalar y mantener.

Cliente-Servidor

Separación de responsabilidades

Separa la interfaz de usuario del procesamiento de datos. El cliente maneja la presentación y el servidor procesa la lógica de negocio y datos. Permite distribución de responsabilidades.

SOA

Servicios orientados a arquitectura

Conjunto de servicios de negocio que colaboran entre sí. Cada servicio es independiente y puede ser reutilizado. Facilita la integración de sistemas heterogéneos.

Microservicios

Servicios independientes

La aplicación se divide en pequeños servicios independientes. Cada servicio maneja una funcionalidad específica. Permite escalabilidad y mantenimiento independiente.

Arquitectura Hexagonal

Puertos y adaptadores

Aísla la lógica de negocio de la tecnología externa. También conocida como arquitectura de puertos y adaptadores. Facilita el testing y la independencia de frameworks.

DDD

Domain-Driven Design

Modela el software para que refleje fielmente el negocio. Domain-Driven Design se enfoca en la lógica de dominio y la comunicación entre expertos técnicos y de negocio.

MVC

Modelo-Vista-Controlador

Patrón que separa la aplicación en Modelo, Vista y Controlador. El Modelo maneja los datos, la Vista la presentación y el Controlador la lógica de negocio.

Repository

Patrón de acceso a datos

Patrón que abstrae la lógica de acceso a datos. Centraliza las operaciones de persistencia y facilita el testing y mantenimiento del código.

Marcos de Arquitectura

Guías y reglas estandarizadas para diseñar y gobernar la arquitectura de software

TOGAF

The Open Group Architecture Framework

Método paso a paso para desarrollar y gestionar la arquitectura a nivel empresarial. Proporciona un enfoque estructurado y completo para la arquitectura empresarial.

Zachman Framework

Matriz de Clasificación Arquitectónica

Matriz para clasificar y asegurar que todos los aspectos de la arquitectura (qué, cómo, quién, dónde, cuándo, por qué) estén cubiertos de manera integral.

C4 Model

Modelo de Visualización en 4 Niveles

Modelo para visualizar la arquitectura de software en 4 niveles de detalle: Contexto, Contenedores, Componentes y Código. Facilita la comunicación entre stakeholders.
